# Security Policy

## Supported Versions

The following versions of Ketchum's Quantum Physics Labs by Quantara are currently supported with security updates.

| Version | Supported |
|----------|----------|
| Main Branch | ✅ |
| Latest Release | ✅ |
| Previous Major Release | ⚠️ Critical Fixes Only |
| Legacy Releases | ❌ |

Only the latest actively maintained release receives regular security updates and patches.

---

## Reporting a Vulnerability

The security of Ketchum's Quantum Physics Labs by Quantara is taken seriously. If you discover a security vulnerability, please report it responsibly and privately.

### How to Report

Please submit vulnerability reports through one of the following methods:

- GitHub Security Advisories (preferred)
- Private email: Evan.ketchum2026@outlook.com

Please include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Affected versions
- Proof-of-concept code (if available)
- Suggested remediation (optional)

---

## Response Timeline

After a report is received:

- Initial acknowledgment: within 72 hours
- Triage and validation: within 7 business days
- Status updates: provided throughout the investigation
- Security patch target: based on severity and complexity

Critical vulnerabilities may receive expedited handling.

---

## Responsible Disclosure

Please do not publicly disclose security vulnerabilities until they have been investigated and a fix has been released.

Researchers who practice responsible disclosure and act in good faith will not be subject to legal action for reporting vulnerabilities.

---

## Scope

This policy applies to:

- Core Quantara platform
- Quantum simulation engines
- QED and QFT modules
- AI research systems
- APIs
- Authentication systems
- Infrastructure components
- Official repositories maintained by the project

Third-party dependencies are subject to their respective security policies.

---

## Security Updates

Security fixes may be released as:

- Patch releases
- Hotfixes
- Emergency security advisories
- Dependency updates

Users are encouraged to keep deployments updated to the latest supported version.

---

## Security Philosophy

Ketchum's Quantum Physics Labs by Quantara follows a security-first development approach emphasizing:

- Secure-by-default architecture
- Least-privilege access controls
- Vulnerability monitoring
- Continuous dependency auditing
- Secure coding practices
- Infrastructure hardening
- Responsible disclosure and transparency
